Version 1.2.1

Saturday, July 07, 2001

 

 

1.    Introduction

This document describes this release of the Mica Graphics Framework.

The information presented herein is:

      Important changes

      Bugs fixes

      Added features

      Next release

2.    Important Changes

2.1.                 Initialization

      MiSystem.init() and MiSystem.init(Applet applet)

      One of these methods MUST be called before any other Mica methods are called. MUST MUST MUST!

2.2.                 Build environment

      Ant

Apache Ant is now used instead of makefiles. See the build directory for the build.xml file and batch files for building on MS Windows.

2.3.                 Runtime environment

      MS Windows

MS Windows is now fully supported

      Solaris

Sun Solaris is no longer our development environment. This is part of a move to using only Windows2000 and Linux for development.

      Applets

Applets are now fully supported. Mica now uses an implementation of the sys/MiiSystemIOManager.java interface to do input and output operations and in particular MiSystemApplicationIOManager.java and MiSystemAppletIOManager.java. These can, of course, be replaced for custom applets (which all applets are because the communication API between client and server side is not standardized)

      Swing

Swing is now supported almost fully (some problems still occur with Swing dialog boxes bugs which should not be encountered by most Mica programmers). Note that Mica should be set to double buffered if it occupies the window with other Swing widgets.

3.    Bug Fixes

3.1.                 General

      JDK Printing

Printing using the JDK API now supports multiple (horizontal and or vertical) pages. There is also a new parts/MiPrintPanel, which replaces the parts/MiExportPanel, and which does a better job of displaying the options available.

      MiZConnectionLayout

This was fixed so that it now displays a Z shape even during connection creation by the user even when the connection must be connected at both ends

      MiARC

This had a number of bugs. All fixed.

      MiColorManager

Fixed a bug having to do with translation of hexadecimal text colors into Colors and back again. Because of the transparency nibble at the top of the color value, this sequence would generate results like #ffffff -> white -> #ffffffff. This is now fixed so that String comparisons between the color specified and the color retrieved match exactly.

      MiAttributes

A number using count is maintained so that MiAttributes are removed from the MiAttributeCache cache when they are no longer being used.

4.    Added Features

4.1.                 General

      Printing Panel

There is a new parts/MiPrintPanel, which replaces the parts/MiExportPanel, and which does a better job of displaying the options available.

      Debug output

Information about any ManagedPointManagers a MiPart may have is now output

      VisibilityToolBar

Now hides connection points instead of making them invisible. This is because hidden graphics parts still occupy geometric space (whereas invisible ones do not) and so a user can connect up to a hidden connection point.

      MiPageManager

The PageColor property has been added so that the edge color, as well as the background/fill color, of a drawing page can be specified.

      MiDrawingGrid

The GridMinPixelSizeVisible property has been added to prevent the condition of having too many grid points displayed when the user zooms way out.

      MiIMoveSelectedObjects.java

This event handler has been added which allows the movement of the selected item(s) in the editors by the user using the up, down, left and right arrow keys.

      MiiModelEntityReference

This interface, and its reference implementation MiModelEntityReference, has been added in order to support libraries of immutable Model Entities that are referenced by mutable instances within the application.

      MiNamedSubPartManagedPoint

This managed point has been added to support points that exist as points of a part that is one part of many in a container. This subpart is specified by name.

      MiFileChooser

This dialog box, and all other areas of Mica, now do I/O though the MiSystem.getIOManager() interface in order to support applets.

      XML Support

The src/xml directory now contains the XML and DOM support classes. SAX2 and JAXP are now supported and should be used instead of SAX1 where possible.

      MiiUnits

Units are now fully supported by Mica and MiPropertyDescription. See com.swfm.util.swfm-util.jar

      MiPropertyDescription

Now supports exclusions that specify, for example, that everything in a range is a valid number except the values specified as excluded.

Now supports inclusions/special cases that specify, for example, that everything in a range is a valid number in addition to the values specified as special cases.

      Images

In AWT single buffered mode, the java.awt.Graphics gets confused clip bounds. A workaround has been implemented and this fixes a number of bugs where images were not being displayed correctly (or more common, not at all, as in MiExample2).
